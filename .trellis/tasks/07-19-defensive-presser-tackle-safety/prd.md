# 防守抢断与禁区安全

## Goal

让 defensive presser 在普通防守和争议球场景中更早、以约 4.0 的小力度直接出脚抢断，减少进攻式调整；同时在己方禁区按更严格的安全条件限制出脚，降低从错误侧触球造成乌龙的风险。

## Background

- 当前 `Player.defensive_press_and_clear()` 在距球不超过 `DEFENSIVE_PRESSER_CLEAR_DISTANCE_M = 0.5m` 时直接出脚，否则直线追球。
- 当前 defensive presser 不绕球、不等待 offensive striker 的球后点或射门对齐，已经是少调整的直接逼抢风格。
- 当前 `Player.plan_defensive_clear()` 始终把高层目标设为对方球门中心，力度为 `DEFENSIVE_PRESSER_CLEAR_POWER = 5.0`。
- 当前出脚只检查机器人到球的距离，没有检查机器人是否位于球的己方球门侧，也没有针对己方禁区或小禁区设置额外安全门槛。
- 成人场地己方禁区范围由场地几何动态确定：从己方门线向场内延伸 3.0m，横向半宽 3.0m；小禁区向场内延伸 1.0m，横向半宽 2.0m。实现不得写死这些场地坐标。
- defensive presser 已与 offensive striker、守门员 CLEAR 和开球力度解耦，本任务应只修改 defensive presser 路径。

## Recommended Strategy

### 禁区外：低力度宽松抢断

- 将直接出脚距离从 0.5m 放宽到约 0.65m。
- 将 defensive presser 出脚力度从 5.0 降至 4.0。
- 仍直接朝球逼抢，不增加 offensive striker 式绕球、球后点或长时间瞄准。
- 增加轻量进入/退出迟滞，例如 0.65m 进入、0.80m 退出，避免机器人在距离边界反复启动和释放踢球。
- 禁区外不增加球后侧安全判断；进入距离后立即使用低力度向场地正前方（`+X`）出脚。

### 己方禁区：安全侧条件出脚

- 不能仅因距离足够近就出脚。
- 安全重点仅检查 presser 是否位于球的己方球门侧，不增加精确方向或角度校准。
- 用户明确要求避免方向校准导致反复调整；因此取消约 40° 的接触方向对齐门槛。
- 不满足安全侧条件时，presser 不踢球，改为直接移动到球的正后方安全点；到达球后侧并进入抢断距离后即可出脚。
- “已位于球后侧”定义为 `presser.x <= ball.x - 0.10m`。
- 不安全时的球后目标为 `(ball.x - 0.35m, ball.y)`，并限制在场地合法范围内。
- 安全判断和禁区边界使用独立 `DEFENSIVE_PRESSER_` 参数，不复用 offensive striker 参数。
- 禁区内出脚方向固定为场地正 X 方向，即只要求把球向前踢离己方球门，不再选择外侧目标或对方球门中心。
- 固定向前方向只用于低力度安全出脚，不引入进攻式瞄准、绕球或角度迟滞。
- 用户确认安全判断只在己方禁区启用；禁区外不受球后侧条件限制。

### 小禁区：同一严格安全条件

- 用户选择较积极方案：小禁区与大禁区使用相同的严格安全条件。
- defensive presser 在小禁区内只要位于球后侧且距离满足门槛，就允许使用低力度向前出脚。
- 不因守门员可用而额外禁止 presser 出脚，但仍保持现有守门员策略和角色分配不变。

## Requirements

1. 防守出脚力度调整到约 4.0，并保持与进攻、守门员和开球独立。
2. 禁区外放宽抢断距离，保持直接逼抢和少调整风格。
3. 己方禁区内必须增加球后侧安全条件，不能仅按距离出脚，但不增加精确方向角度校准。
4. 不安全时应继续封堵或移动到球的安全侧，不能沿用旧踢球命令。
5. 禁区和小禁区范围必须从 `context.field` 计算。
6. 所有新增阈值集中放在 `src/param.py`，使用 `DEFENSIVE_PRESSER_` 前缀。
7. 保持 ATTACKING、DEFENDING、CONTESTED 判断、角色选择、保护球员、守门员六状态、M-05、readiness、重启和路径规划不变。

## Out of Scope

- offensive striker 射门参数或动态力度。
- 守门员 CLEAR 方向和力度。
- 开球、传球、T07.1、T08-T11。
- 新攻防模式或无关重构。

## Confirmed Safety Geometry

- 球后侧最小纵向余量：0.10m。
- 不安全时球后安全点距离：0.35m。
- 不要求横向精确对齐。
- 大禁区和小禁区使用相同规则。

## Confirmed Behavior

- defensive presser 所有低力度抢断均固定向场地正前方（`+X`）出脚。
- 禁区外只检查进入/退出距离，不增加球后侧安全条件。
- 己方大禁区和小禁区均增加球后侧安全条件。
- 禁区内不满足安全条件时移动到球后 0.35m 的安全点，满足 `presser.x <= ball.x - 0.10m` 后即可出脚。

## Acceptance Criteria

- [ ] 禁区外 defensive presser 能在约 0.65m 内以约 4.0 力度直接抢断。
- [ ] 禁区外不因球后侧条件或方向对齐产生额外等待。
- [ ] 距离边界附近不会频繁启动和释放出脚动作。
- [ ] defensive presser 不执行进攻绕球或射门对齐流程。
- [ ] 己方禁区内从错误侧接近球时不会随意出脚。
- [ ] 己方禁区内位于球后侧后可直接向场地正前方低力度出脚，不等待精确方向校准。
- [ ] 小禁区与大禁区使用同一套球后侧安全条件，不额外禁止 presser 出脚，也不要求精确角度校准。
- [ ] defensive presser 调整不影响 offensive striker、守门员 CLEAR 或开球力度。
- [ ] M-05 仍不覆盖正在逼抢、封堵或安全出脚的 presser。
- [ ] 仅列出手动验证场景，不运行自动验证命令。
