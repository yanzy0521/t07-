# 核对赛场规则与代码风险

## Goal

以 `o:\Robot\赛场规则.md` 为最高规则依据，只读核对当前 T07 代码，识别明确违规、潜在判罚风险、实现缺口和需要仿真确认的行为。

## Background

当前代码为用户恢复的 T07 基线，包含比赛阶段分派、异常机器人过滤、临时守门员、普通攻防三态和六状态守门员策略。用户已提供可读的 Markdown 规则副本，并明确要求以其为准。

## Requirements

- 提取规则文件中与技术合规、比赛状态、开球、定位球、避让距离、触球开放、机器人数量、守门员、判罚和动作限制有关的条款。
- 记录关键条款的规则编号、原意和适用比赛阶段。
- 理解并追踪当前代码的裁判数据入口、Phase 分派、可用机器人过滤、职责分配、重启行为、普通攻防和守门员状态机。
- 将每项规则映射到具体代码路径，区分符合、部分符合、明确不符合和静态阅读无法确认。
- 按严重程度列出当前风险，并说明可能触发的比赛后果或判罚。
- 只读分析；不修改源代码、配置或策略实现。

## Constraints

- 指定 Markdown 规则文件是最高依据，不用旧任务、过时文档或一般经验覆盖其明确条款。
- 不运行 build、test、lint、type-check、format、仿真、部署、IDE diagnostics 或运行时验证。
- 不创建 commit，不 push。

## Acceptance Criteria

- [x] 已读取指定规则文件并为关键结论提供规则编号依据。
- [x] 已说明代码主要入口及规则敏感的数据流和状态流。
- [x] 已形成规则条款到代码实现的逐项映射。
- [x] 已按明确违规、高风险、实现不完整和需仿真确认分类风险。
- [x] 未修改项目实现，未运行任何被禁止的验证命令。
