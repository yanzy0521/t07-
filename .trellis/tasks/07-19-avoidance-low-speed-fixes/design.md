# 技术设计

## 修改边界

仅修改：

- `src/player.py`：全局 heading 与局部 VFH 的安全回退、累计路径 lookahead。
- `src/utils/path_planner.py`：对角 corner 约束和连续边碰撞检查。

不修改参数、调用方、障碍收集、守门员策略和重启目标生成。

## 控制数据流

1. `walk_to()` 收集障碍并运行 A*。
2. A* 成功后，按累计路径长度选取 waypoint。
3. 计算全局 waypoint heading 及其局部 ray clearance。
4. 若该 clearance 小于 `PLAN_CLEARANCE`，运行现有 `_plan_heading()`。
5. 只有局部候选的 clearance 严格大于全局 heading clearance 时才使用局部候选；否则保留全局方向。
6. 后续距离、净空、近距/远距和最终限速公式保持不变。

这样不会无条件抛弃全局路径，也不会在局部规划没有改善时制造额外路径抖动。

## 累计路径 lookahead

从当前 pose 开始，按 `path[1:]` 的 segment 顺序累计消耗剩余 lookahead：

- 当前 segment 足以覆盖剩余距离时，在该 segment 内插值返回。
- 否则扣除当前 segment 长度并继续下一段。
- 路径总长不足 lookahead 时返回最终点。

继续跳过 `path[0]`，因为它是量化后的 start grid point，强制机器人回到该点可能产生不必要的回头动作。

## A* 连续边几何

### 普通边

将当前栅格点到邻居栅格点视为线段。若线段到任一障碍中心的最短距离不大于 `obs.radius + GLOBAL_OBSTACLE_MARGIN_M`，该边不可用。

### 对角边

除连续边检查外，还要求对应的水平和垂直 cardinal 邻居均不 blocked，避免斜穿两个 blocked cell 的角。

### blocked start 脱困

连续边检查不能简单应用于起点位于障碍内的第一步，否则所有离开线段都因起点已相交而被拒绝。

第一步采用受限例外：

- 找出包含 start 的膨胀障碍集合。
- 邻居终点必须比 start 更远离每一个重叠障碍中心。
- 对未包含 start 的其他障碍，仍使用正常连续边碰撞检查。
- 邻居终点仍必须是 free。

这不是完整运动学脱困器，但可避免“新增连续检查后必然锁死”，同时不会允许第一步继续深入已重叠障碍。

## 兼容性与取舍

- 不更改公开方法签名。
- 不更改速度公式和参数，因此正常无障碍行走速度不变。
- A* 可能比以前更频繁返回 `None`，随后沿现有 VFH 回退；这是阻止无效穿障路径的预期变化。
- 不处理 blocked goal、球门障碍策略边界和跨帧路线迟滞；这些需要独立设计或仿真依据。
- 不改变对方重启目标的 1.6m 投影，但路径本身当前仍未把 1.6m 规则圆建模成规划障碍，本任务不扩大到该规则修正。
