# 修复避障持续低速

## Goal

以最小、可静态确认的修改，减少全局 A* 返回路径后因局部 heading 净空过低而持续慢行或停止，并修复路径跟随和栅格邻接中明确的不连续几何问题。

## Background

静态审计确认：A* 允许靠近障碍膨胀边缘的路径，而速度层会把相同方向压到很低；A* 成功后当前实现不会因第一 heading 净空不足回退局部 VFH。此外，路径 lookahead 按单段而非累计路径长度选择，A* 对角移动也未禁止穿过 blocked corner。

## Requirements

- 当 A* waypoint heading 的净空低于局部规划安全阈值时，比较局部 VFH 候选；只有局部候选净空更大时才覆盖全局 heading。
- 局部 VFH 覆盖全局 heading 时，不把全局 waypoint 继续显示为实际控制点，避免可视化误导。
- `_path_waypoint()` 按路径累计长度消耗 `GLOBAL_PATH_LOOKAHEAD_M`，不再把多段短路径当成零成本跳过。
- A* 对角邻接必须检查两个相邻 cardinal 栅格点，禁止从 blocked corner 中斜穿。
- A* 栅格边必须做连续线段与膨胀障碍圆的相交检查；起点已在障碍内时，仅允许第一步朝所有重叠障碍的外侧移动，并仍不得穿入其他障碍。
- 不修改障碍半径、净空速度阈值、守门员参数、重启 1.6m 目标投影或战术调用开关。
- 不实施跨目标转向状态重置；其显著目标变化阈值和动态目标语义需要仿真依据，当前不属于高把握修改。

## Constraints

- 不清理、回退或覆盖工作区已有修改。
- 不运行 build、test、lint、type-check、format、仿真、部署、开发服务器、IDE diagnostics 或任何运行时验证命令。
- 仅修改与上述几何和回退逻辑直接相关的源文件；不顺带重构旧 `detour()` 或调整策略参数。

## Acceptance Criteria

- [ ] A* heading 净空不足时，局部 VFH 可以在净空确实更优的情况下接管控制方向。
- [ ] `_path_waypoint()` 的 lookahead 预算跨路径 segment 累计消耗。
- [ ] A* 不再允许两个相邻 cardinal 栅格被挡住时直接走对角线。
- [ ] 普通 A* 栅格边不再仅靠端点 free 就穿过膨胀障碍圆。
- [ ] blocked start 有受限的向外脱困第一步，不因连续边检查被无条件锁死。
- [ ] 未改变当前速度阈值、障碍半径、守门员和对方重启规则参数。
- [ ] 未运行项目禁止的任何验证命令。
