# 守门员阻挡与大脚解围优化

## Goal

提升守门员在防守阶段的实战表现，使其能够更主动地封堵来球路线，并在禁区/门前危险区域拿到球权或具备解围机会时执行大脚解围，降低门前丢球风险。

## Requirements

- 检查现有守门员程序中阻挡、出击、控球、踢球/解围相关逻辑，定位守门员“不太会阻挡”和“不大脚解围”的主要原因。
- 守门员在球接近我方球门或处于高危区域时，应优先移动到球与球门之间的有效封堵位置，而不是只追球或停留在无效位置。
- 守门员具备安全踢球条件时，应倾向于将球大脚解围到远离我方球门的方向，避免软踢、误踢回门前或长时间带球。
- 解围逻辑必须兼顾比赛状态和安全边界，避免在不应踢球的状态下违规触球，或在门前近距离把球踢向自家球门。
- 尽量复用项目中已有的运动、踢球、角色策略工具，避免新增与现有策略重复或冲突的控制逻辑。

## Acceptance Criteria

- [x] 已定位守门员策略入口和影响阻挡/解围行为的关键条件。
- [x] 守门员在防守危险场景中会选择更合理的封堵点，体现“挡在球和球门之间”的行为。
- [x] 守门员在可踢球且需要解围时会使用明显更强的清球动作，将球送离我方门前。
- [x] 改动不会破坏已有角色分配、普通防守、界外球/球门球等特殊比赛状态逻辑。
- [x] 代码改动保持可读、局部、可回退，并记录关键策略阈值的意图。
