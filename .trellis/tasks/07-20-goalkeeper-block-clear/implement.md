# 守门员阻挡与大脚解围优化 - Implementation Plan

## Checklist

1. 阅读项目相关 Trellis 规范，确认代码风格和策略层约定。
2. 定位守门员程序入口、角色分配入口、踢球/移动辅助函数和比赛状态判断。
3. 分析当前守门员阻挡失败与不大脚解围的条件缺口。
4. 修改守门员策略：
   - 增强危险球识别。
   - 增强球门线封堵点计算。
   - 增强安全大脚解围分支。
5. 检查改动是否影响特殊比赛状态、角色分配和已有防守逻辑。
6. 按项目规则不运行构建、测试、lint、仿真或 IDE 诊断，由用户手动验证。

## Manual Verification Suggestions

用户可在仿真或实机中重点观察：

- 对方带球/射门进入我方禁区时，守门员是否站到球与球门之间。
- 球停在我方门前或禁区内时，守门员是否主动大脚解围。
- 守门员解围方向是否远离我方球门，避免踢回门前。
- 界外球、球门球、开球等特殊状态下是否没有异常抢踢。

## Verification Constraint

根据本项目用户规则，本任务不由 AI 运行 build、test、lint、type-check、format、仿真或其他验证命令。
