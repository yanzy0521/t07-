# 守门员阻挡与大脚解围优化 - Design

## Scope

本任务聚焦守门员单角色策略：危险球判断、门线/球门前封堵站位、可踢球时的解围决策，以及与现有比赛状态/角色调度的兼容性。不重写全队攻防框架，不调整无关角色的常规策略。

## Technical Approach

1. 定位守门员策略入口，确认当前守门员在不同比赛状态下的职责分支。
2. 梳理现有坐标系、我方球门方向、射门/清球力度接口和禁区边界常量。
3. 对阻挡逻辑做局部增强：
   - 高危球时优先计算球到我方球门中心线段上的封堵点。
   - 封堵点限制在守门员合理活动范围内，避免越界或离门过远。
   - 在球已经非常接近且可触球时允许更主动出击。
4. 对解围逻辑做局部增强：
   - 满足比赛状态允许、距离允许、方向安全时，使用大力度踢球。
   - 解围方向优先指向远离我方球门的前场/边路安全区域。
   - 避免朝自家球门、门前中心或危险区域回传。

## Compatibility and Safety

- 保留现有特殊状态判断，尤其是界外球、球门球、开球、暂停等场景。
- 不引入新的全局状态机，避免与已有角色分配冲突。
- 阈值应集中、命名清晰，便于现场调参。
- 如果现有项目已有清球/射门辅助函数，优先复用，避免重复实现底层踢球控制。

## Rollback Shape

所有改动应尽量集中在守门员策略文件或其直属辅助函数中；若实战效果不佳，可以通过回退该文件中的封堵点计算和解围分支恢复原行为。
