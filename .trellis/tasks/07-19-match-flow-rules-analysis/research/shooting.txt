追球者选择算法的思路
追球者选择由`DefaultPlaybook.select_chaser()`实现，每帧执⾏⼀次。它是⼀个三层筛选过程：

Chaser每帧只能是⼀个⼈——只有⼀个球员会执⾏追踢⾏为，其余⼈都是⽀援站位Supporter 。
第⼀层：ReadySlot资格过滤
select_chaser的第⼀步是资格筛选，筛选掉完全不适合成为Chaser 的机器⼈。
筛选和球员默认位置ReadySlot 联系紧密。ReadySlot 决定了球员在⽐赛Ready阶段所站的位
置，也是球员的兜底策略会回到的位置。
ReadySlot 这是⼀个枚举类型，定义在 src/soccer_framework/types.py 。共有三个位
置：KEEPER, SIDE, CENTER 。
这三个ReadySlot的差异贯穿了追球者选择算法和之后追球者⾏为逻辑：
-CENTER：球队主攻⼿。追球代价最低（优先追球），踢球时⾛完整决策链（传→射→带）。
-SIDE：边路辅助。只在条件合适时追球，踢球时⾛"只传不射⻔"策略——不许射⻔。
-KEEPER：⻔将。正常情况不追球，只有当球进⼊⼰⽅危险区域（禁区范围）时才出击解围。

代码块
def _slot_can_challenge( 
    self, 
    slot: ReadySlot, 
    ctx: PlayContext, 
) -> bool: 
    targeting = self.kit.targeting 
    ball = ctx.known_ball 
    if slot == ReadySlot.KEEPER: 
        return targeting.ball_in_own_defensive_area(ball) 
    if slot == ReadySlot.SIDE: 
        return targeting.side_should_challenge(ctx) 
    return True

调整测试
我们可以通过测试⼀个极端的select_chaser 策略来观察和默认策略的效果差异。
使⽤以下函数代替⽰例select_chaser ，将会始终选择站位第⼆靠后的球员成为CHASER 。
代码块
def select_chaser(self, context: PlayContext) -> int: 
 
        config = self.kit.config 
 
        scored: list[tuple[float, int]] = [] 
        for player_id in config.player_ids: 
            robot = context.teammates.get(player_id) 
            if robot is None or robot.pose is None: 
                continue 
            scored.append((robot.pose.x, player_id)) 
 
        if not scored: 
            return min(config.player_ids) 
 
        scored.sort() 
        return scored[1][1] 
经过观察，我们发现⼀旦chaser带球跑到球队最前位置就会站住。同时，中间位置的球员会⽴刻被分
配chaser。这个策略的问题可以怎样调整呢？你可以尝试优化这种策略。
