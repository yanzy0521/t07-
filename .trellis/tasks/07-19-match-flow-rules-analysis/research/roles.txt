为什么要分配⻆⾊
在机器⼈⾜球⽐赛中，场上状态（球的位置、球员状态、⽐分等）瞬息万变。⻆⾊分配要解决的问题
是：在当前赛场动态下，教练怎样安排球员在场上⾏动的思考⽅式？⻆⾊即对应了球员⼀种特定的思
考⽅式。
基于球在场地上的位置分配⻆⾊是⼀种简单直观地分配⽅案，⽐如下⾯的分配策略：
球的位置 位置最靠后的球员 位置第⼆靠后的球员 位置第三靠后的球员
场地前1/3 守⻔员 进攻⻆⾊ 进攻⻆⾊
场地中间1/3 守⻔员 防守⻆⾊ 进攻⻆⾊
场地后1/3 守⻔员 防守⻆⾊ 防守⻆⾊\n\n--- PAGE 2 ---\n根据球位置分配⻆⾊⽰意图-可拖动查看效果.html
使⽤这种⻆⾊分配策略，当球和对⽅球⻔距离较近时，会分配更多的进攻⻆⾊，攻势更猛。
也可以基于球员和球的距离设计⻆⾊分配策略，⽐如：
距离球最近的球员 距离球第⼆近的球员 距离球第三近的球员
进攻⻆⾊ 防守⻆⾊ 守⻔员
分配⻆⾊的本质是安排球员做更正确的事。
由于仿真赛中机器⼈配置相同，且不受电池、电机的运⾏状况影响，位姿（位置和朝向）是机器⼈间
最主要的区别，也是分配⻆⾊时考虑的⾸要因素。评价位姿要充分考虑赛场环境：即使⼀个球员和球
的距离最近且朝向球，但如果有对⼿在该球员和球之间，它也可能并不适合作为唯⼀的追球球员。
好的⻆⾊分配策略会充分利⽤场上的所有信息，包括⽐分信息。如果⽐赛快要结束，你还落后，请分
配进攻⻆⾊给所有球员，放⼿⼀搏吧！

理解Role
Playbook 是教练视⻆，职责是输出⼀个映射表（如 {Player_1: "chaser", Player_2: 
"defender"} ）；Role 是球员视⻆，是具体⻆⾊的抽象定义。每个 Role 都对应⼀棵专职的⾏
为⼦树（Sub-tree ）。当球员被分配 Role 时，它就会跑这棵⼦树⾥的具体叶⼦节点⾏为（如找
位、插上、拦截）。
⽰例策略中预设的⻆⾊有三个：
⻆⾊ 职责 场上⾏为 ⼦树结构
chaser 追球、控球、射⻔ 始终朝向球移动，进⼊踢
球范围后射⻔
Selector→KickBranch|MoveToTarget
supporter ⽀援、接应、传球 站在球的侧后⽅，准备接
应
MoveToTarget
goalkeeper 防守球⻔、禁区解围 站在⻔前，球进禁区时出
击
Selector→KickBranch|MoveToTarget
none 静⽌（扩展） 兜底 WaitForBall\n\n--- PAGE 6 ---\n预设分配逻辑

def assign_roles(self, ctx: PlayContext) -> RoleAssignment: 
 
    chaser_id = self.select_chaser(ctx) 
    goalkeeper_id = self._configured_goalkeeper() 
 
    mapping: dict[int, str] = {} 
    for player_id in self.kit.config.player_ids: 
        if player_id == goalkeeper_id: 
            mapping[player_id] = ROLE_GOALKEEPER 
        elif player_id == chaser_id: 
            mapping[player_id] = ROLE_CHASER 
        else: 
            mapping[player_id] = ROLE_SUPPORTER 
 
    return RoleAssignment(mapping) 
DefaultPlaybook 分配⻆⾊的逻辑是
1. 调⽤select_chaser ⽅法找到最适合成为chaser ⻆⾊的球员id，在配置⽂件中找到默认的守
⻔员id。
2. 给队伍中最适合的球员分配⻆⾊GAOLKEEPER和CHASER 
3. 给队伍中其他⼈分配SUPPOTER 

代码块
def assign_roles(self, ctx: PlayContext) -> RoleAssignment: 
    ball = ctx.ball 
    player_ids = self.kit.config.player_ids 
    all_attack = ball.x > 0 
    mapping = {pid: ROLE_CHASER if all_attack else ROLE_GOALKEEPER for pid in 
player_ids} 
    return RoleAssignment(mapping) 
这段代码中，使⽤PlayContext.ball.x 获取球的纵向坐标。x>0意味着球在对⽅半场，分配进攻
⻆⾊ROLE_CHASER 给全部球员，否则分配ROLE_GOALKEEPER 给全部球员。你可以尝试此策略和
预设策略的对⽐效果（/src/play/playbook.py )。
总结
在设计与优化策略分配时，请牢记以下要点：
1. Playbook.assign_roles全权负责宏观⻆⾊的动态分配。
2. RoleAssignment是Playbook.assign_roles的计算结果。结果写⼊⾏为树⿊板，供⾏为树节点读
取。
3. 策略每帧重新计算，⻆⾊可能在毫秒级重新分配。
4. 考虑少⼈、受罚等场上最极端边界情况，确保系统具备兜底能⼒。
