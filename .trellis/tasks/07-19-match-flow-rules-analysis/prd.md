# 理解比赛规则并确认当前代码完整比赛流程

## Goal

以官方比赛规则为最高依据，完整梳理虚拟足球机器人比赛从准备到结束的状态流转，并结合当前 Demo 代码确认程序实际实现的完整比赛流程，产出一份可供后续策略优化前使用的 Markdown 分析文档。

## Background and Constraints

- 用户已提供官方比赛规则 PDF、官方程序解析与优化教程 PDF，以及项目内已有代码解析文档作为资料来源。
- 官方比赛规则是最高依据；官方优化教程和既有代码解析只可作为辅助材料，不能替代规则。
- 本阶段只做阅读、理解、核对和文档产出。
- 禁止修改程序代码、设计新策略、启动仿真或运行验证命令。
- 对无法从规则或代码确认的内容，必须明确列为待确认问题，不得猜测。

## Requirements

- 阅读并整理官方比赛规则中一场比赛可能经历的完整过程，包括赛前准备、开球、正常比赛、进球后重启、出界、任意球、犯规处罚、暂停继续、结束及其他特殊状态。
- 阅读完整 Demo 代码，定位程序启动、初始化、比赛状态接收、状态判断、球权与重启类型判断、角色/任务分配、进攻/防守/守门员/定位球逻辑、动作执行、循环更新、暂停和结束处理等关键入口。
- 按比赛实际发生顺序说明代码模块之间如何连接、切换，而不是仅做单文件或单函数摘要。
- 建立规则状态与代码实现的对应关系，逐项列出规则事件、代码判断入口、文件/类/函数/行为树节点/配置项、主要逻辑、结束条件、共同处理模块。
- 将实现状态区分为：明确实现、通过通用逻辑间接实现、当前代码未找到明确实现。
- 核对项目中已有代码解析文档是否准确、完整、仍与当前 Demo 一致，并指出遗漏、误解、局部化分析或缺少依据之处。
- 在项目中生成 Markdown 分析文档，包含用户指定的 10 项内容。

## Acceptance Criteria

- [ ] 分析文档引用具体资料、文件路径、类名、函数名、行为树节点、配置项或关键代码位置。
- [ ] 文档包含官方规则下的完整比赛流程和当前代码主运行流程。
- [ ] 文档包含规则状态与代码模块对应表，并标明明确实现、间接实现、未找到实现。
- [ ] 文档包含各类比赛重启与特殊情况处理流程。
- [ ] 文档包含对已有代码解析文档的核对结论。
- [ ] 文档列出所有存在歧义或需要用户确认的问题。
- [ ] 未修改任何程序代码，未开始策略优化。
- [ ] 未运行 build、test、lint、type-check、format、simulation、deployment、dev server 或其他验证命令。

## Out of Scope

- 程序代码修改。
- 策略优化或新策略设计。
- 编写或运行自动化验证。
- 将官方优化教程中的建议当作当前代码已实现行为。

## Open Questions
