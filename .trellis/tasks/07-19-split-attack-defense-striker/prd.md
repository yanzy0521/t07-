# 分离进攻与防守抢球动作

## Goal

规划并指导后续 AI 将普通进攻的射门角色与普通防守/争议球的逼抢角色彻底解耦，使进攻射门可以独立调得更积极，而不改变防守逼抢和解围行为。

## Background

当前普通比赛先统一分配 `primary_attacker`，随后在 `DEFENDING` 或 `CONTESTED` 中重新选择离球最近的 `challenge_player`。防守选择结果还会写回 `store.normal_attacker`，因此会影响后续进攻主攻的粘滞选择。

进攻球员通过 `Player.attack()` 使用绕球、球后点、踢球进入/退出距离和普通踢球力度。防守逼抢虽然直接追球，没有调用完整 `attack()`，但在解围时调用同一个 `Player.plan_kick()`，从而复用普通进攻的踢球方向和 `KICK_POWER_DEFAULT` / `KICK_POWER_BACKFIELD`。因此调整进攻射门参数仍可能改变防守解围。

## Requirements

- 将普通进攻角色明确命名和建模为 offensive striker，仅用于 `ATTACKING` 的主攻、搭档临时接管以及当前我方定位球的进攻回退。
- 将 `DEFENDING` 和 `CONTESTED` 的近球机器人明确建模为 defensive presser，不再称为 striker/primary attacker。
- 进攻 striker 和防守 presser 使用独立的跨帧角色粘滞状态，防守选择不得写回进攻 striker 状态。
- 为防守 presser 提供独立选择迟滞，只有另一名场上机器人明显更近时才交换逼抢职责。
- 将进攻追球、绕球、射门距离、对齐角度、射门方向和射门力度参数归入独立的进攻参数组。
- 防守 presser 使用独立的追球/解围入口、解围距离、解围方向和解围力度，不调用进攻射门规划器。
- 保留底层 `Player.kick()` 作为共享硬件动作，但隔离进攻与防守的高层规划和跨帧动作状态。
- 攻防职责切换时清理不适用于新职责的绕球和踢球迟滞，避免进攻状态泄漏到防守或反向泄漏。
- 保持守门员六状态、重启安全、M-05 无活动预防、少人降级和普通双人进攻结构不变。
- 仅生成给新窗口 AI 的实施提示词；本任务不修改源代码。

## Out of Scope

- 不调整具体进攻激进参数数值，只建立可独立调节的参数边界。
- 不重新设计射门目标选择、传球、T07.1 或 T08-T11。
- 不改变普通攻防模式判定。
- 不修改守门员解围逻辑。

## Acceptance Criteria

- [x] 已确认当前角色状态和踢球规划存在进攻/防守耦合。
- [x] 已定义 offensive striker 与 defensive presser 的职责边界。
- [x] 已要求两者拥有独立角色迟滞、动作入口和参数组。
- [x] 已限制范围，避免改变守门员、重启和后续固定战术。
- [x] 已生成可供新窗口 AI 直接实施的完整提示词。
