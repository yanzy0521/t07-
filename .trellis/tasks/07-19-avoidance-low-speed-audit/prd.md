# 审计避障持续低速根因

## Goal

仅通过静态阅读当前工作区代码，判断机器人开启避障后偶发持续低速是否属于合理的近障减速、到达制动或原地转向，还是来自路径规划、净空语义、障碍膨胀、路径点选择、速度缩放、跨帧状态或调用方目标冲突，并输出带准确文件行号的根因排序与最小修改建议。

## Background

用户观察到机器人开启避障走位时，有时步速会被压得非常慢。本次审计必须从 `Player.walk_to()` 的一次调用完整追踪到最终 `vx / vy / vyaw`，同时核对全局 A*、路径跟随、无全局路径时的局部 VFH 回退、障碍收集和膨胀、近距全向与远距弧线状态、原地转向、到达制动、跨帧状态以及主要战术调用方。

## Requirements

- 至少阅读 `src/player.py`、`src/param.py`、`src/utils/path_planner.py`、`src/utils/obstacles.py`、`src/utils/geom.py`，以及 `src/main.py` 中 `walk_to()`、`move_to_position()`、`guard()`、`_walk_to_restart_target()` 的全部调用。
- 若当前代码包含 M-05 无活动预防走位，只检查其移动目标与避障参数是否可能触发低速，不扩展到其他策略功能。
- 完整写出近距全向和远距弧线两条速度生成链，覆盖目标距离、目标方向、障碍收集、A* 成败、waypoint 选择、最终 heading、selected clearance、距离缩放、净空缩放、近距模式、原地转向、heading factor、`MAX_LINEAR` 与 `set_velocity()` 最终限幅。
- 核对净空值是中心距还是边缘净空，以及 `SAFETY_MARGIN`、`GLOBAL_OBSTACLE_MARGIN_M`、`PLAN_CLEARANCE` 和各障碍半径是否存在重复膨胀或语义混杂。
- 给出 `selected_clearance` 为 0.05m、0.10m、0.20m、0.35m、0.65m 时的静态净空缩放示例。
- 比较 A* blocked 几何与速度层 `_heading_clearance()` 几何，检查栅格边缘贴行、线段检查、对角切角、平滑安全性，以及 blocked start/goal 对起步和脱困的影响。
- 检查 `_path_waypoint()` 的选择依据、每帧重规划、路径方向震荡、lookahead、可视化一致性，以及对原地转向和 `heading_factor` 的影响。
- 检查 `_near_walk_mode`、`_turn_in_place` 的迟滞与状态切换，区分合理原地转向、参数保守和可能卡滞。
- 检查到达制动使用最终目标还是局部 waypoint 距离，并判断动态站位目标是否可能让机器人长期处于制动区。
- 列出球、队友、对手、门柱和球网的实际有效半径及参数叠加语义。
- 检查球门障碍是否在 `avoid_ball` 或 `avoid_robots` 开启时被统一加入，并重点判断守门员、READY、重启和球门附近目标是否容易被持续压速。
- 检查机器人起点位于障碍范围内时，全局规划、局部规划和净空缩放是否存在“越需要脱困，平移越被压为零”的死锁。
- 检查最多 2 秒的新鲜度、动态障碍旧位姿、每帧重规划、路径侧选择迟滞，以及初始化但未实际参与当前规划的跨帧状态。
- 建立主要战术动作的球避障、机器人避障和球门障碍调用矩阵，并识别目标位于球附近但仍避球的冲突场景。
- 将结论严格区分为合理行为、参数过于保守、实现缺陷和只能通过仿真确认四类，不提前下结论。

## Acceptance Criteria

- [ ] 报告包含结论摘要和最可能的前三个根因。
- [ ] 报告完整列出近距全向与远距弧线速度公式，并指出所有可能把速度压到接近零的乘数或状态。
- [ ] 风险清单逐项包含严重程度、分类、准确文件行号、触发条件、减速机制以及持续低速或卡死可能性。
- [ ] 参数表覆盖所有相关参数、当前值、实际作用和可能问题。
- [ ] 调用场景矩阵覆盖用户指定的攻击、接应、防守、守门员、READY、重启和 M-05 场景。
- [ ] 修改建议按优先级给出最小方案、安全性影响、对重启 1.6m 避让的影响和后续仿真需求，但不实施修改。
- [ ] 报告给出后续手动仿真应采集的字段。
- [ ] 报告明确说明未修改源代码，且未运行任何被禁止的验证命令。
