# 排查拼抢前异常后撤

## Goal

仅通过静态阅读当前最新代码，确认进攻和防守球员在准备开大脚或与对手拼抢时为何经常先向后撤离球，而不是直接出脚，并给出准确调用链、触发条件、根因排序和最小修改建议。

## Background

用户观察到普通进攻与防守球处理动作存在共同现象：机器人接近球后倾向先后撤一段距离，再尝试踢球或继续接近。当前代码近期新增了球处理职责状态和独立的进攻射门、防守抢断逻辑，必须基于最新工作区重新追踪。

## Requirements

- 检查 `src/player.py` 中进攻追球、绕球、球后点、踢球迟滞、球处理职责、防守抢断和 `walk_to()` 调用。
- 检查 `src/main.py` 中普通进攻、防守逼抢、争议球、搭档临时接管和角色切换调用链。
- 检查 `src/param.py` 中所有相关距离、角度、绕球、踢球和防守抢断参数。
- 区分机器人主动后撤目标、绕球圆弧的后向分量、目标/职责每帧切换、踢球触发条件不满足、后端踢球前置条件和避障影响。
- 分别写出进攻与防守从策略调用到最终行为的完整状态链。
- 给出最可能根因排序和只做高把握修改时的建议，但不实际修改代码。

## Constraints

- 不修改源代码、配置或产品文档；除本 Trellis 任务记录外不写入项目文件。
- 不运行 build、test、lint、type-check、format、仿真、部署、开发服务器、IDE diagnostics 或任何运行时验证命令。
- 不清理、回退或覆盖工作区已有修改。

## Acceptance Criteria

- [ ] 已明确进攻是否因球后点或绕球状态主动后撤。
- [ ] 已明确防守是否也调用相同后撤逻辑，或由独立抢断阈值/状态切换造成。
- [ ] 已检查最新球处理职责状态是否导致进攻、防守动作互相重置或切换。
- [ ] 所有主要结论包含准确文件行号和触发条件。
- [ ] 已区分代码可确认原因与需仿真确认因素。
- [ ] 未修改源代码，未运行任何禁止的验证命令。
