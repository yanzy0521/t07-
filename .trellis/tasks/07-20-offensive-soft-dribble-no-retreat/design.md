# 进攻小力带球减少后撤 - Design

## Scope

本任务只处理进攻球员在普通进攻中的持球、追球、绕球和出脚逻辑。防守逼抢、后场防守解围、守门员解围、保护站位和定位球不在本任务范围内。

## Technical Approach

1. 从 `Player.attack()` 和普通比赛进攻入口定位 offensive striker 的子动作。
2. 检查当前后撤来源：是否来自球后位 `_behind_ball()`、绕球状态机、直接射门角度门槛或干扰球分支。
3. 在进攻参数区新增独立的小力带球参数，保持与防守参数隔离。
4. 在进攻球处理逻辑中增加一个保守的 soft-dribble 分支：
   - 球位于己方/中场或尚未形成明确射门机会。
   - 机器人离球足够近。
   - 朝对方球门方向推球不会向后场移动。
   - 当前没有必要为了完美球后位继续后撤。
5. 保留正常射门优先级：角度和位置合适时仍按现有射门力度踢。

## Non-Goals

- 不修改 `DEFENSIVE_PRESSER_*`。
- 不修改 `GOALKEEPER_*`。
- 不修改防守保护站位参数。
- 不改定位球、界外球或开球逻辑。

## Rollback Shape

若小力带球效果不好，可回退新增进攻参数和 `Player.attack()` 内新增分支，防守逻辑无需回退。
