# TODO: 球状态估计、短时预测与卡尔曼滤波

## 当前决定

本轮不修改球数据结构、ROS 输入或 Runtime，也不加入卡尔曼滤波。

当前仿真输入已经是场地坐标系中的球位置真值，且置信度固定为 `1.0`。直接对真值做完整滤波可能增加控制滞后，并可能把裁判摆球、世界重置或合法瞬移错误识别为异常跳点。

只有出现以下需求之一时再实施：

- 需要根据球速拦截移动球；
- 仿真球位置更新频率或时间抖动影响追球；
- 接入带噪声、置信度和短时遮挡的实机视觉数据；
- 需要在很短的数据间断期间保持连续追球目标。

## 建议实施顺序

### 1. 区分新观测和重复快照

控制循环可能以 30Hz 重复读取同一条球观测。估计器只能在 `BallState.last_seen_at` 变化时执行观测更新；其余控制帧只能做保持或预测。

禁止把每个控制帧都当成一条新观测，否则重复位置会把估计速度错误地压向零。

### 2. 增加基础观测保护

- 拒绝 NaN 和无穷坐标；
- 检查时间倒退和过长观测间隔；
- 长时间断流后重置历史；
- 对速度进行合理限幅；
- 对低速结果设置小死区，减少方向抖动。

跳点拒绝必须支持“权威重置”例外。裁判摆球、仿真世界重置、场地方向切换或数据源重启时，应直接用新真值重新初始化，不能继续保持旧轨迹。

### 3. 先实现简单球速估计

优先使用场地坐标系位置和真实观测时间估计 `vx`、`vy`，并进行有限的平滑和限幅。不要从机器人坐标系或相机坐标系直接差分，否则机器人自身运动会污染球速。

建议保留这些不同语义：

- `last_seen_at`: 最后一条真实观测的时间；
- `estimated_at`: 当前估计位置所代表的时间；
- `is_predicted`: 当前位置是否仅由预测得到；
- `vx`、`vy`: 场地坐标系球速。

### 4. 只允许短时预测

短时断流时可以用恒速模型预测约 `0.25-0.4s`。超过该时间应返回球未知，而不是继续沿旧轨迹预测，也不应沿用当前新鲜度窗口进行长时间外推。

策略层必须能区分“真实观测位置”和“仅预测位置”，避免把预测球直接当成可踢球依据。

### 5. 有必要时再加入卡尔曼滤波

推荐状态为：

```text
[x, y, vx, vy]
```

采用恒速状态转移，观测量为 `[x, y]`。滤波器应放在平台无关的 Runtime 世界模型层，保持：

```text
WorldSnapshot.ball（原始观测）
    -> BallEstimator
    -> Context.ball（估计结果）
```

不要把估计状态分散到 ROS 回调、策略和每个 `Player` 中。

## 置信度注意事项

实机 Demo 的部分参数使用 `0-100` 置信度范围，当前项目使用 `0-1`，且仿真真值固定为 `1.0`。后续若迁移置信度加权观测噪声，必须统一采用当前项目的 `0-1` 语义，不能直接复制原阈值。

## 暂不实施的内容

- 加速度预测；
- 两秒以上的恒速轨迹；
- 队友球共享和融合；
- 独立 ROS 球预测节点；
- 在没有可靠球速前实现移动球拦截。
